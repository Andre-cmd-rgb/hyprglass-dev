# Hyprglass V1.2 production review

## Verdict

V1.2 is a production-hardening pass over V1.1. The important fix is not visual. It is process ownership: Settings must never hard-kill and respawn Waybar from inside a terminal session, because a respawned Waybar can inherit the Settings terminal session and die when that terminal closes. V1.2 fixes that path.

## Fixed from the Arch VM report

- Settings no longer hard-restarts Waybar through `pkill -x waybar` during normal preference apply.
- Settings now asks Waybar to reload through `SIGUSR2`, then only starts Waybar if it is missing.
- Detached app launches from Waybar now use `setsid -f kitty ...`, so clicking Settings in the bar does not create a fragile parent/child relationship with Waybar.
- Detached process spawning in Go now uses a new session and redirects stdio to `/dev/null`.
- The normal Settings screen is simpler: Appearance, Display/Keyboard, Network/Modem, Audio, Power, Update, Developer options.
- Repair, Doctor, Services, wallpaper repair, raw display/keyboard panels, and CachyOS hardware tools are hidden under Developer options.

## Installer review

The installer now has a cleaner split:

- first setup handles hardware/session setup;
- Settings handles day-to-day configuration;
- Developer options handles diagnostics and repair.

First setup now covers:

- theme mode;
- accent color;
- keyboard layout/variant;
- display scale;
- enabling NetworkManager, Bluetooth, ModemManager, and power profile services;
- optional LTE/5G modem APN and SIM PIN autounlock service;
- optional CachyOS `chwd -a` hardware auto-configuration through `--auto-hardware` or an explicit first-setup prompt on CachyOS.

The installer still does not silently replace kernels or convert Arch into CachyOS. That is correct.

## CachyOS support review

CachyOS support remains explicit and safe:

- `/etc/os-release` auto-detection stays in place;
- `--distro=cachyos` forces the CachyOS package profile;
- `--rate-mirrors` calls `cachyos-rate-mirrors` only on CachyOS;
- `--auto-hardware` calls `chwd -a` only on CachyOS and only when requested;
- Settings exposes CachyOS system actions under Developer options, not in the main user UI.

## Versioning review

Versioning is now file-based:

- `VERSION` is the canonical version file;
- `install.sh`, `scripts/build.sh`, `scripts/check.sh`, and `Makefile` read `VERSION`;
- the Go binary receives the version through `-ldflags`.

## Verification added

The check script now fails if:

- `VERSION` is missing or malformed;
- Waybar click commands do not detach Settings with `setsid -f kitty`;
- Settings hard-kills Waybar instead of using the reload path;
- current hyprpaper wallpaper block syntax regresses;
- Waybar CSS regresses to 8-digit hex alpha;
- installed config does not rewrite hyprpaper with an absolute wallpaper path.

## Remaining honest limits

The sandbox cannot validate pacman package availability, a real Hyprland session, actual Waybar rendering, real LTE/5G hardware, Bluetooth hardware, display EDID behavior, fingerprint readers, or CachyOS `chwd` side effects. Those are correctly marked as warnings/skips, not claimed as fully tested.
