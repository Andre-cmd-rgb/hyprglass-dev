# Hyprglass V1.3 icon/font production review

## Problem

On a clean Arch install, the top bar could render several Waybar icons as square placeholder glyphs. The previous package profile installed `ttf-jetbrains-mono-nerd`, but did not install the dedicated Nerd Font symbols fallback package. The Waybar CSS also did not include all common installed family names used by GTK/Pango/fontconfig for JetBrains Mono Nerd Font and Symbols Nerd Font.

## Root cause

Waybar renders text through GTK/Pango. Private-use Nerd Font glyphs used for Ethernet, Bluetooth, muted audio, brightness, battery, settings, and power only render when fontconfig can resolve a font containing those symbols. If the session resolves to a normal monospace fallback, the bar shows tofu/square placeholders.

## Fixes in V1.3

- Added `ttf-nerd-fonts-symbols-mono` to Arch and CachyOS package profiles.
- Added `fontconfig` explicitly to Arch and CachyOS package profiles.
- Kept `ttf-jetbrains-mono-nerd` as the main text/icon font.
- Expanded Waybar CSS font fallback stack:
  - `JetBrainsMono Nerd Font`
  - `JetBrainsMonoNL Nerd Font`
  - `JetBrainsMono Nerd Font Propo`
  - `JetBrainsMono Nerd Font Mono`
  - `Symbols Nerd Font Mono`
  - `Symbols Nerd Font`
  - `Noto Sans`
  - `DejaVu Sans`
- Added module-specific font fallback for icon-heavy modules.
- Added `hyprglass icons status`.
- Added `hyprglass icons repair`.
- Added Settings → Developer options → Icon/font repair.
- Installer now runs `ensure_icon_fonts` during normal installs and during `--update` to repair older installs that skipped the new symbols package.
- Installer refreshes fontconfig cache with `fc-cache -f`.
- Doctor now checks `fc-match` resolution for JetBrains Mono Nerd and Symbols Nerd Font Mono.
- Verification now fails if package profiles, Waybar CSS, installer font repair, or CLI icon repair hooks are missing.

## User-facing behavior

Normal Settings stays simple. Icon/font repair is hidden under Developer options because users should not need it after a correct install. Direct CLI repair remains available for broken sessions:

```sh
hyprglass icons repair
hyprglass settings apply
```

## Verification

`./scripts/check.sh` passes all repository checks available in the sandbox. The final state is WARN only because the sandbox lacks shellcheck, pacman, and a live Hyprland session.
